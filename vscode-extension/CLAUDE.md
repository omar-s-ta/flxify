# Flxify VS Code Extension - Project Knowledge

## What Is This?

A VS Code extension (currently v0.4.0) that brings all 110 Flxify text transformation scripts into the editor. Users select text, press Cmd+Shift+B, pick a script from a QuickPick menu, and the text is transformed in place. Scripts are shared ("universal") between the web app and the extension — the same `.js` files work in both contexts. Published at https://marketplace.visualstudio.com/items?itemName=flxify.flxify

## Architecture

### The Bridge Pattern

The core challenge: scripts were written for the web app's BoopState API (which uses CodeMirror). The extension creates a **BoopState bridge** that maps the same API to VS Code's editor:

| Script calls... | Web app does... | Extension does... |
|----------------|-----------------|-------------------|
| `state.text` | Read/write CM selection or doc | Read/write VS Code selection or doc |
| `state.postError(msg)` | DOM toast notification | `vscode.window.showErrorMessage()` |
| `state.postInfo(msg)` | DOM toast notification | `vscode.window.showInformationMessage()` |
| `state.insert(text)` | CM dispatch insert | `editBuilder.replace()` |

### Require Shim

Scripts use `require('@flxify/moduleName')` to import lib modules. The extension replicates this with a CommonJS sandbox using `new Function()`. This is safe in Node.js (no CSP restrictions like the browser).

The shim:
1. Strips `@flxify/` prefix and `.js` suffix
2. Reads the module source from `src/scripts/lib/`
3. Wraps it in a CommonJS sandbox (exports, module)
4. Caches the result

**Gotcha:** `lodash.boop` has a dot in its name. The shim handles `require('@flxify/lodash.boop')` correctly.

### Dynamic Script Loading

Scripts are NOT hardcoded. The extension reads `src/scripts/` at activation:
1. `fs.readdirSync()` discovers all `.js` files
2. Metadata is parsed from `/** { ... } **/` blocks
3. Trailing commas are stripped before JSON.parse
4. Execute functions are created via `new Function('require', 'state', source + ...)`
5. Scripts are sorted alphabetically by name

### Multi-Cursor Support

The command iterates `editor.selections` inside a single `editor.edit()` call. This means:
- Each cursor/selection gets its own BoopState instance
- All replacements happen atomically
- One Cmd+Z undoes the entire operation

## Key Gotchas

1. **Scripts run in non-strict mode.** Some use implicit globals like `buf = ""`. Do NOT add `'use strict'` to the Function wrapper.

2. **Metadata trailing commas.** Script JSON blocks may have trailing commas. The parser uses `.replace(/,\s*([\]}])/g, '$1')` to handle this.

3. **`new Function()` is fine here.** Unlike the web app (CSP-restricted), VS Code extensions run in Node.js where `new Function()` is unrestricted. This is the simplest way to load scripts dynamically without requiring them as Node modules.

4. **Scripts are excluded from TypeScript compilation.** `tsconfig.json` excludes `src/scripts` — they're plain JS loaded at runtime.

5. **`Cmd+Shift+B` conflicts with VS Code's default "Run Build Task".** The keybinding has `"when": "editorTextFocus"` to scope it, but users may want to customize this.

6. **Icon must be 128x128 PNG.** The marketplace rejects other sizes. Generated from the web app's logo.png via sips crop+resize.

## Development Workflow

- **Edit extension logic:** Modify `src/extension.ts`, then `npm run compile`
- **Add/edit scripts:** Edit files in `src/scripts/`, reload VS Code (scripts are loaded at activation)
- **Test:** Press F5 to launch Extension Development Host
- **Package:** `npm run package` (runs `vsce package --allow-missing-repository`)
- **Publish:** `npx @vscode/vsce publish --allow-missing-repository` (requires Azure PAT, already logged in)

## Syncing Scripts with Web App

Scripts in `src/scripts/` are copies from the web app's `scripts/` directory. To sync:
```bash
cp ../scripts/*.js src/scripts/
cp ../scripts/lib/*.js src/scripts/lib/
```

There is no automatic sync — this is intentional to keep the extension self-contained.

### Full Publish Workflow (after adding/editing scripts)
1. Copy scripts: `cp ../scripts/*.js src/scripts/ && cp ../scripts/lib/*.js src/scripts/lib/`
2. Bump version in `package.json` (e.g., 0.3.0 → 0.4.0)
3. Compile: `npm run compile`
4. Publish: `npx @vscode/vsce publish --allow-missing-repository`
5. Git commit and push from the root repo

### Gotchas Learned During Publishing

7. **Azure DevOps PAT must be "All accessible organizations".** If scoped to a specific org, the marketplace API rejects it with "Access Denied: needs permission to View user permissions on a resource". Always select "All accessible organizations" when creating the PAT.

8. **Publisher must exist before `vsce login`.** Create the publisher at marketplace.visualstudio.com/manage first. The same Microsoft account must be used for both the publisher and the Azure DevOps PAT.

9. **`.vscodeignore` must exclude aggressively.** Without proper exclusions, `vsce package` bundles everything (node_modules, pkg/, .git/, etc.) resulting in 20000+ files. Only ship: `out/`, `src/scripts/`, `icon.png`, `README.md`, `CHANGELOG.md`, `LICENSE`, `package.json`. Must also exclude `bin/**` (Go toolchain binaries like `dlv` ~18MB and `gopls` ~37MB can inflate the VSIX by ~55MB).

10. **Stray `pkg/` directory breaks TypeScript compilation.** If a Go toolchain GOPATH overlaps with the extension directory, TypeScript picks up `.ts` files from `pkg/mod/`. Fix: add `"include": ["src/**/*.ts"]` and `"pkg"` to tsconfig.json exclude.

11. **Icon must be exactly 128x128 PNG.** Generated from web app's logo.png via: `sips -c 768 768 logo.png --out icon.png && sips -z 128 128 icon.png`. The crop centers the image before resizing.

12. **`Cmd+Shift+B` overrides VS Code's "Run Build Task".** The keybinding uses `"when": "editorTextFocus"` to scope it, but power users may still conflict. Document how to rebind via Keyboard Shortcuts.

13. **Generator scripts must use `state.insert()`, not `state.text =`.** Scripts that generate new content (UUID Generator, Lorem Ipsum, New Boop Script) must check `state.isSelection` before writing. If text is selected, use `state.text = result` to replace the selection. If no text is selected, use `state.insert(result)` to insert at the cursor position. Using `state.text = result` with no selection replaces the entire document, which destroys existing editor content. Transform scripts (Upper Case, Sort Lines, Format JSON, etc.) should continue using `state.text = result` because they read and transform the full text.

14. **`--allow-missing-repository` flag is required for publishing.** The extension's `package.json` references a GitHub repo that may not have the extension as a standalone repo. Without this flag, `vsce publish` may warn or fail about the missing repository.

15. **Version must be bumped before each publish.** The marketplace rejects re-publishing the same version. Always increment the version in `package.json` before running `vsce publish`. Follow semver: patch (0.3.0 → 0.3.1) for script additions, minor (0.3.0 → 0.4.0) for features, major for breaking changes.

16. **`bin/` and `pkg/` directories must be removed from git tracking.** Go toolchain binaries in `bin/` (~56MB: `dlv`, `gopls`) and Go module cache in `pkg/` (~349MB, 21,449 files) may be tracked by git. Remove with `git rm --cached -r bin/ pkg/` from the vscode-extension directory. The root `.gitignore` covers `vscode-extension/bin/` and `vscode-extension/pkg/`.

17. **`.claude/` directory should not be in the public repo.** Claude Code local settings (`.claude/settings.local.json`, agent definitions) are developer-specific. Remove from git with `git rm --cached -r .claude/` at the repo root.
